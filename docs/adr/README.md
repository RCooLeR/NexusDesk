# Architecture Decision Records

Date: 2026-05-28

This folder contains architecture decision records for NexusDesk. Use ADRs for decisions that affect architecture boundaries, security policy, persistence formats, connector behavior, generated artifact formats, packaging/distribution, or extension/plugin execution.

## Index

No ADRs have been accepted yet. Add new accepted or proposed records here as they are created.

## Naming

Use a four-digit sequence and short kebab-case title:

```text
0001-example-decision.md
```

## Template

```markdown
# ADR NNNN: Short Title

Status: Proposed | Accepted | Superseded

Date: YYYY-MM-DD

## Context

What problem, constraint, or tradeoff forced this decision?

## Decision

What are we choosing?

## Consequences

What improves, what gets harder, and what risks remain?

## Validation

How will we know the decision works in code, tests, docs, packaging, or user workflows?

## Follow-Ups

- [ ] Concrete follow-up task.
```
